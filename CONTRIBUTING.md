# 기여 가이드

## 문서 작성 규칙

- **언어**: 한국어. 명령어, 코드, 기술 용어 원문은 영어 유지
- **코드블록**: 언어 태그 필수 (```bash, ```json, ```markdown 등)
- **대상 독자**: 비엔지니어 포함 — 기술 용어는 간단히 풀어서 설명
- **업데이트 날짜**: 각 문서 하단에 `**마지막 업데이트**: YYYY-MM-DD` 포함
- **분량**: 핵심 위주로 간결하게. 불필요한 서론이나 반복 설명 지양

## 디렉토리 구조

```
docs/          → 사용자 가이드 (설치, 활용법, 관리법)
reference/     → 기능별 상세 레퍼런스 (CLAUDE.md, Skills, Hooks 등)
skill/         → Claude Code Skill 정의
```

- `docs/`는 "어떻게 시작하고 활용하는가"에 초점
- `reference/`는 "기능이 정확히 어떻게 동작하는가"에 초점
- 새 문서 추가 시 적절한 디렉토리에 배치

## 문서 번호 체계

`docs/`와 `reference/`는 하나의 연속된 번호 체계를 공유합니다:

| 번호 | 위치 | 문서 |
|------|------|------|
| 01 | docs/ | 시작하기 |
| 02 | reference/ | CLAUDE.md 작성법 |
| 03 | reference/ | Skills |
| 04 | reference/ | Hooks |
| 05 | docs/ | 프롬프팅 패턴 |
| 06 | reference/ | MCP Servers |
| 07 | docs/ | 컨텍스트 관리 |
| 08 | reference/ | IDE 통합 |
| 09 | reference/ | Agent Teams |
| 10 | docs/ | 활용 가이드 |

새 문서 추가 시 기존 번호 사이에 끼워넣기보다, 다음 번호(11~)를 사용하세요.

---

**마지막 업데이트**: YYYY-MM-DD
```

## 컨벤션 강제 방법

이 저장소는 두 가지 방법으로 문서 컨벤션을 강제합니다.

### 1. `.claude/rules/doc-conventions.md` — 자동 강제

Claude Code가 `docs/` 또는 `reference/` 의 마크다운 파일을 수정할 때 자동으로 로드되는 규칙입니다. Claude가 문서를 편집할 때 컨벤션을 자동으로 따르게 합니다.

별도 설정 없이 이 저장소에서 Claude Code를 실행하면 자동 적용됩니다.

### 2. `/doc-review` Skill — 수동 검토

문서 수정 후 컨벤션 준수 여부를 검토하는 스킬입니다.

```
# 특정 파일 검토
/doc-review docs/05-prompting-patterns.md

# 변경된 파일 전체 검토
/doc-review
```

검토 결과가 항목별 Pass/Fail 테이블로 출력되며, 위반 사항이 있으면 구체적인 수정 내용을 알려줍니다.

## PR 체크리스트

- [ ] 한국어 작성, 코드/명령어는 영어 유지
- [ ] 코드블록에 언어 태그 포함
- [ ] 하단에 `마지막 업데이트` 날짜 갱신
- [ ] README.md의 문서 목록에 반영 (새 문서인 경우)
- [ ] `/doc-review`로 컨벤션 검토 통과
