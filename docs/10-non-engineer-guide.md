# Claude 활용 가이드

> Claude는 코딩 도구가 아닙니다. 업무를 자동화하고, 코드를 작성해서 더 복잡한 자동화도 할 수 있는 **업무 도구**입니다.

---

## 가장 중요한 것

### Claude 사용법은 Claude에게 물어보세요

이 가이드를 전부 외울 필요 없습니다. **Claude 자체가 가장 좋은 가이드**입니다.

```
Claude Code 사용법 알려줘
이 프로젝트에서 뭘 할 수 있어?
/compact가 뭐야?
Connector 어떻게 연결해?
```

모르는 게 있으면 Claude에게 바로 물어보세요. Claude는 자기 자신의 기능을 잘 알고 있습니다.

### 반복 작업은 "Skill"로 자동화할 수 있습니다

**Skill**이란? 자주 하는 작업을 `/명령어` 하나로 실행할 수 있게 만든 **나만의 단축키**입니다. 엑셀의 매크로, 카카오톡의 빠른 답장 템플릿과 비슷합니다.

예를 들어 매주 월요일에 주간 리포트를 만든다면:

```
내가 매주 월요일에 하는 주간 리포트 작성 작업을 Skill로 만들어줘
```

Claude가 자동으로 Skill 파일을 생성하고, 이후에는 `/weekly-report` 한 줄로 실행됩니다. **코딩 지식이 없어도** Claude에게 요청하면 알아서 만들어줍니다.

## Claude로 할 수 있는 것

Claude는 세 가지 방식으로 업무를 도와줍니다:

| 방식 | 설명 | 예시 |
|------|------|------|
| **대화** | 질문하고 답변 받기 | "이 데이터 트렌드를 분석해줘" |
| **자동화** | 반복 업무를 Claude가 대신 처리 | Slack 메시지 요약, 파일 정리, 리포트 생성 |
| **코드 작성** | 필요하면 코드를 짜서 더 복잡한 자동화 | 데이터 파이프라인, 대시보드, 배포 스크립트 |

## Claude 제품군 이해하기

### Claude Desktop — 일상 업무의 AI 파트너

Claude Desktop은 데스크톱 앱으로, 대화·문서 작성·분석 등 일상 업무에 활용합니다.

### Cowork — "결과물을 만들어주는" 에이전트

**Cowork**는 Claude Desktop 안에서 동작하는 기능으로, 단순 대화를 넘어 **실제 결과물을 만들어줍니다**.

할 수 있는 것:
- 엑셀 스프레드시트 (수식, 여러 탭 포함)
- 파워포인트 프레젠테이션
- 정리된 문서와 리포트
- 파일 정리 및 구조화
- 데이터 분석 및 시각화

**사용 방법:**
1. Claude Desktop에서 원하는 결과를 설명
2. Claude가 계획을 세우고 실행
3. 중간중간 진행 상황을 보여주고, 수정 요청 가능
4. 완성된 결과물이 파일로 저장됨

**예시:**
```
이번 달 매출 데이터가 들어있는 엑셀 파일을 분석해서,
주간별 트렌드 차트와 인사이트 3가지를 포함한
경영진 보고용 파워포인트를 만들어줘.
```

**비유**: "일을 시키고 자리를 비워도 되는 인턴". 결과물이 준비되면 알려줍니다.

**요구사항**: Claude Pro, Max, Team, Enterprise 구독 필요. macOS/Windows 데스크톱 앱 전용.

### Connector — 외부 도구와 연결

**Connector**는 Claude를 여러분이 이미 사용하고 있는 도구들과 연결해줍니다. 한 번 연결하면 Claude가 해당 서비스의 데이터를 읽고, 작업을 수행할 수 있습니다.

**주요 Connector 목록:**

| 카테고리 | 서비스 | 할 수 있는 것 |
|----------|--------|--------------|
| **커뮤니케이션** | Slack, Gmail, Microsoft 365 | 메시지 요약, 이메일 초안 작성, 알림 관리 |
| **프로젝트 관리** | Asana, Linear, Jira, Monday.com | 이슈 생성, 진행 상황 확인, 태스크 관리 |
| **콘텐츠** | Notion, Google Drive, WordPress | 문서 검색, 페이지 생성, 파일 접근 |
| **디자인** | Canva, Figma | 디자인 참조, 에셋 관리 |
| **데이터** | GitHub, Hex, Amplitude | 코드 검색, 데이터 분석, 지표 조회 |
| **재무** | Stripe | 결제 데이터 조회 |

**설정 방법:**
1. Claude Desktop 또는 claude.ai에서 대화창의 "+" 버튼 클릭
2. "Manage connectors" 선택
3. 원하는 서비스 선택 → "Connect" 클릭
4. 해당 서비스 로그인 및 권한 승인

**추가 비용 없음** — 기존 구독에 포함.

**예시:**
```
Slack #marketing 채널의 이번 주 대화를 요약해줘.
중요한 의사결정과 액션 아이템을 정리해서 Notion에 페이지로 만들어줘.
```

### Claude Code — 로컬 환경을 자유롭게 활용하는 에이전트

Claude Code는 터미널/IDE에서 실행되며, 여러분의 컴퓨터에서 **거의 모든 작업**을 수행할 수 있습니다.

코드 수정에 특화되어 있지만, 그 외에도:
- **로컬 파일 조작**: 폴더 정리, 파일 이름 변경, CSV 변환, 이미지 리사이즈
- **자료 검색 및 분석**: 프로젝트 내 코드/데이터 검색, 패턴 분석, 의존성 파악
- **명령어 실행**: git, npm, 배포 스크립트, DB 쿼리, API 호출
- **문서 생성**: README, API 문서, 변경 이력, 리포트 자동 작성
- **자동화 스크립트**: 반복 작업을 코드로 만들어 영구적으로 자동화

---

## Claude Desktop vs Cowork vs Claude Code

| | Claude Desktop | Cowork | Claude Code |
|---|---|---|---|
| **접근 방법** | 데스크톱 앱에서 대화 | Claude Desktop 내 에이전트 | 터미널, IDE, 데스크톱 앱 |
| **결과물** | 텍스트 답변 | 파일(엑셀, PPT, 문서 등) | 코드 수정, 파일 조작, 명령어 실행 |
| **Connector** | 사용 가능 | 사용 가능 | MCP 서버로 연결 |
| **코딩 지식** | 불필요 | 불필요 | 불필요 (자연어로 지시) |
| **적합한 용도** | 질문, 문서 작성, 분석 | 리포트, 프레젠테이션, 파일 정리 | 코드 수정, 로컬 파일 조작, 자료 검색, 자동화, 배포 |

**어떤 걸 쓸까?**
- 빠른 질문/답변 → **Claude Desktop**
- 결과물이 파일로 필요할 때 → **Cowork**
- 로컬 환경에서 직접 작업할 때 → **Claude Code** (코드 수정, 파일 정리, 검색, 자동화 등)

## 직군별 실전 활용 사례

### PM (프로덕트 매니저)

**Cowork로 리포트 자동 생성:**
```
지난 분기 사용자 피드백 데이터를 분석해서
주요 이슈 Top 5와 개선 제안을 포함한
경영진 보고용 파워포인트를 만들어줘.
```

**Connector로 프로젝트 현황 파악:**
```
Linear에서 이번 스프린트의 진행 상황을 확인하고,
완료된 이슈와 지연된 이슈를 정리해줘.
```

**Claude Code로 기술 스펙 작성:**
```
현재 코드에서 결제 로직이 어떻게 구현돼 있는지 분석하고,
멤버십 자동 업그레이드 기능의 기술 스펙을 작성해줘.
```

### 디자이너

**Connector로 디자인 참조:**
```
Figma에서 현재 디자인 시스템의 컴포넌트 목록을 가져와서
사용 빈도와 함께 정리해줘.
```

**Cowork로 컴포넌트 문서화:**
```
디자인 시스템의 색상 토큰, 타이포그래피, 스페이싱 규칙을
정리된 문서로 만들어줘.
```

### 마케터

**Cowork로 데이터 분석 + 시각화:**
```
이 CSV 파일의 캠페인 성과 데이터를 분석해서
채널별 ROI 비교 차트와 인사이트를 포함한 엑셀 리포트를 만들어줘.
```

**Connector로 커뮤니케이션 자동화:**
```
Gmail에서 지난 주 고객 문의 메일을 분류해서
주제별 요약을 Slack #cs-team 채널에 공유해줘.
```

**Claude Code로 트래킹 자동화:**
```
현재 추적 중인 이벤트 목록을 코드에서 추출하고,
누락된 이벤트가 있으면 추가해줘.
```

### 상품 MD (머천다이저)

**Cowork로 상품 분석 리포트:**
```
이 엑셀의 상품별 판매 데이터를 분석해서
카테고리별 매출 순위, 재고 회전율, 마진율을 포함한
주간 MD 리포트를 만들어줘.
```

**Connector로 경쟁사 & 시장 조사:**
```
Google Drive의 '경쟁사 분석' 폴더 자료를 참고해서
우리 주요 카테고리별 경쟁사 가격대와 프로모션 현황을 정리해줘.
```

**Claude Code로 상품 데이터 정리:**
```
상품 CSV 파일에서 카테고리별 평균 판매가, 할인율, 재고 현황을
정리해서 대시보드용 요약 테이블을 만들어줘.
```

**일상 업무 자동화 예시:**
```
이번 시즌 신규 입고 예정 상품 리스트를 바탕으로
상품 소개 카피를 각각 3가지 톤(프리미엄, 캐주얼, 감성)으로 작성해줘.
```

```
지난 달 반품률 상위 10개 상품의 공통점을 분석해줘.
카테고리, 가격대, 리뷰 평점, 배송 방식별로 패턴이 있는지 확인해줘.
```

**Cowork로 보고서 자동화:**

## Interview 패턴 — 요구사항을 구체화하는 방법

Claude에게 "인터뷰해달라"고 하면 단계적으로 질문하며 요구사항을 정리해줍니다.

```
나는 멤버십 등급 자동 업그레이드 기능을 기획하고 있어.
나에게 인터뷰해줘 — 등급 기준, 혜택 차이, 알림 방식,
엣지 케이스에 대해 충분히 파악될 때까지 질문하고,
끝나면 완전한 기획 스펙을 정리해줄래.
```

---

## 효과적인 작업 방법: Research → Plan → Execute

Claude에게 일을 시킬 때는 **한 번에 다 시키지 말고**, 단계를 나눠서 진행하면 훨씬 좋은 결과를 얻습니다.

### 1단계: Research

먼저 Claude에게 현재 상황을 파악하게 합니다.

**마케터 예시:**
```
우리 회사의 현재 이벤트 트래킹 현황을 조사해줘.
어떤 이벤트를 수집하고 있고, 누락된 게 있는지 정리해줘.
```

**PM 예시:**
```
현재 멤버십 등급 체계를 조사해줘.
등급 종류, 각 등급의 혜택, 업그레이드 조건을 정리해줘.
```

**운영팀 예시:**
```
지난 달 고객 문의 데이터를 분석해줘.
문의 유형별 건수와 평균 해결 시간을 정리해줘.
```

이 단계에서 Claude가 파악한 내용을 읽고, 맞는지 확인하세요.

### 2단계: Plan

조사 결과를 바탕으로 계획을 세우게 합니다.

**마케터 예시:**
```
조사 결과를 바탕으로 누락된 이벤트 추가 계획을 세워줘.
어떤 이벤트를 추가할지, 네이밍은 어떻게 할지 정리해줘.
아직 실행하지 말고 계획만 보여줘.
```

**PM 예시:**
```
멤버십 자동 업그레이드 기획안을 작성해줘.
대상 사용자, 업그레이드 기준, 알림 방식, 예외 케이스를 포함해줘.
아직 최종본은 아니니 초안만 보여줘.
```

**운영팀 예시:**
```
문의 유형별로 FAQ를 업데이트하는 계획을 세워줘.
어떤 항목을 추가/수정/삭제할지 정리해줘.
아직 실행하지 말고 계획만 보여줘.
```

계획을 검토하고, 수정이 필요하면 피드백합니다.

### 3단계: Execute

계획이 확정되면 실행을 시킵니다.

```
이 계획대로 진행해줘.
```

### 왜 이 방식이 좋은가?

| 방식 | 문제점 |
|------|--------|
| "이벤트 트래킹 정리해줘" (한 번에) | Claude가 추측으로 진행, 방향이 틀릴 수 있음 |
| Research → Plan → Execute (단계별) | 각 단계에서 검증하므로 결과물 품질이 높음 |

**핵심**: Claude가 틀린 방향으로 오래 달리는 것보다, 중간중간 확인하며 진행하는 게 빠릅니다.

## Claude의 핵심 기능 이해하기

직접 설정하지 않더라도 알아두면 Claude를 더 효과적으로 활용할 수 있습니다.

### CLAUDE.md — 프로젝트의 "업무 매뉴얼"

프로젝트 폴더에 있는 설정 파일로, Claude에게 "이 프로젝트에서는 이렇게 일해"라고 알려줍니다.
- 세션 시작 시 자동으로 읽힘
- `/init` 명령어로 자동 생성 가능

> 더 알고 싶다면: [CLAUDE.md 작성법 상세](../reference/02-claude-md.md) — 스코프, 베스트 프랙티스, rules 시스템

### Skills — 반복 작업의 "단축키"

자주 반복하는 작업을 `/skill-name` 형태로 한 번에 실행합니다.
- 예: `/claude-guide hooks 설정법`
- 팀에서 만든 Skill을 공유하면 누구나 같은 워크플로우 사용
- Skill을 만들고 싶으면 Claude에게 "이 작업을 Skill로 만들어줘"라고 요청

> 더 알고 싶다면: [Skills 상세](../reference/03-skills.md) — SKILL.md 구조, 프론트매터 옵션, 번들 스킬 목록

### Hooks — Claude 동작의 "자동 규칙"

Claude가 특정 행동을 할 때 **자동으로 실행되는 규칙**입니다.
- 예: 파일 수정 시 자동 포맷팅, PR 전 빌드 검증, 응답 완료 시 알림

> 더 알고 싶다면: [Hooks 상세](../reference/04-hooks.md) — 라이프사이클 이벤트, 설정법, 실전 예시

### MCP Servers — Claude Code에서 외부 도구 연결

Claude Code에서는 Connector 대신 **MCP(Model Context Protocol)** 서버로 외부 도구를 연결합니다.
- GitHub, Slack, Figma, Notion 등 연결 가능
- 엔지니어가 프로젝트에 설정해두면 자동으로 사용 가능

> 더 알고 싶다면: [MCP Servers 상세](../reference/06-mcp-servers.md) — 설정법, 주요 서버 목록, 트러블슈팅

### Settings & Permissions — Claude의 "권한 설정"

| 모드 | 동작 | 언제 사용 |
|------|------|----------|
| default | 매번 확인 요청 | 처음 사용할 때 (가장 안전) |
| plan | 계획을 보여준 후 승인하면 실행 | 큰 변경을 검토하고 싶을 때 |
| auto | 안전한 작업은 자동 승인 | 익숙해진 후 빠르게 작업할 때 |

> 더 알고 싶다면: [Settings & Permissions 상세](../reference/05-settings-permissions.md) — 4-tier 설정 계층, 패턴 문법

### Agent Teams — 여러 Claude가 함께 일하기

여러 에이전트가 각자 역할을 맡아 병렬로 작업하는 기능입니다.

## 자주 묻는 질문

**Q: 코딩을 전혀 몰라도 사용할 수 있나요?**
A: 네. Claude Desktop + Cowork는 코딩 지식 없이 사용 가능합니다. Claude Code도 자연어로 지시하면 됩니다.

**Q: Cowork와 Claude Code의 차이는?**
A: Cowork는 Claude Desktop 안에서 파일(엑셀, PPT 등)을 만들어줍니다. Claude Code는 터미널/IDE에서 프로젝트 코드를 직접 다룹니다. 둘 다 자연어로 지시합니다.

**Q: Connector는 추가 비용이 드나요?**
A: 아니요. 기존 구독(Pro, Max, Team, Enterprise)에 포함되어 있습니다.

**Q: Connector로 연결한 데이터는 안전한가요?**
A: 모든 데이터 전송은 암호화됩니다. 본인이 접근 권한이 있는 콘텐츠만 동기화됩니다.

**Q: 실수로 파일을 망가뜨리면?**
A: Claude는 변경 전에 확인을 요청합니다. `/rewind`로 이전 상태로 되돌릴 수도 있습니다.

**Q: Skills이나 MCP는 제가 직접 설정해야 하나요?**
A: 보통 엔지니어링 팀이 미리 설정합니다. 추가 설정이 필요하면 `/claude-guide`로 방법을 물어보세요.
