# Claude Code 컨텍스트 관리 가이드

> Claude의 사용량 규칙을 이해하고, 컨텍스트를 효율적으로 관리하세요.

## 세션과 사용량 초기화 규칙

Claude의 사용량 제한은 **두 가지 시간 단위**로 관리됩니다.

### 5시간 윈도우 (세션 한도)

**첫 번째 메시지를 보낸 시점**부터 5시간 단위로 사용량이 리셋됩니다. 자정이나 캘린더 기준이 아닙니다.

- 오전 10시에 첫 메시지 → 오후 3시에 리셋
- 한도에 도달하면 다음 윈도우 시작 시간이 표시됨
- **피크 시간**(PT 오전 5시~11시 / KST 오후 9시~다음날 오전 3시)에는 한도가 더 빨리 소진될 수 있음

### 주간 한도 (7일 롤링)

5시간 윈도우와 별개로 **7일 단위 총 사용량 한도**가 있습니다.

- 주간 한도에 도달하면 5시간 기다려도 복구되지 않음
- 7일 롤링 윈도우가 리셋되어야 복구됨

### 컨텍스트 윈도우 초기화 (세션 내)

사용량 한도와 별개로, **한 세션 안의 컨텍스트 윈도우**(200K 토큰)도 관리가 필요합니다.

| 상황 | 컨텍스트 초기화? | 설명 |
|------|-----------------|------|
| `/clear` 실행 | **초기화됨** | 모든 대화 기록 삭제, 새로 시작 |
| `/compact` 실행 | **부분 초기화** | 대화를 요약하여 공간 확보 |
| 새 세션 시작 (`claude` 재실행) | **초기화됨** | 새로운 컨텍스트로 시작 |
| `claude -c` (이전 대화 이어가기) | **유지됨** | 이전 대화 그대로 이어감 |
| 자동 압축 (Auto-Compaction) | **부분 초기화** | 시스템이 자동으로 요약 |

### 초기화되어도 사라지지 않는 것

- **CLAUDE.md** — 매번 디스크에서 다시 로드
- **Auto Memory** — 이전 세션에서 배운 내용이 자동 저장
- **.claude/rules/** — 프로젝트 규칙 파일들

---

## 컨텍스트 윈도우란?

Claude는 한 번에 볼 수 있는 정보의 양에 제한이 있습니다. 이를 **컨텍스트 윈도우**라고 부르며, 약 **200,000토큰** (200K 토큰)을 지원합니다.

**비유**: 작업 책상 위에 펼칠 수 있는 서류의 양이 정해져 있다고 생각하세요. 책상이 가득 차면 기존 서류를 정리해야 새 서류를 놓을 수 있습니다.

### 세션 시작 시 자동 로드되는 것

| 항목 | 크기 | 설명 |
|------|------|------|
| 시스템 프롬프트 | ~4,200 토큰 | Claude의 기본 동작 규칙 |
| Auto Memory | 0.5~25K 토큰 | 이전 세션에서 배운 내용 |
| 환경 정보 | ~280 토큰 | 작업 디렉토리, OS, git 브랜치 |
| MCP 도구 목록 | ~120 토큰 | 연결된 외부 도구 목록 |
| CLAUDE.md | 다양 | 프로젝트 설정 파일 전체 |

**팁**: CLAUDE.md는 매번 전체가 로드되므로 200줄 이내로 유지하세요. 자세한 작성법은 [CLAUDE.md 상세](../reference/02-claude-md.md)를 참고하세요.

## 사용량 확인하는 방법

### Claude Code (터미널/데스크톱 앱)

| 명령어 | 용도 |
|--------|------|
| `/context` | 컨텍스트 사용량을 컬러 그리드로 시각화 + 최적화 제안 |
| `/cost` | 이번 세션의 입력/출력 토큰 수와 비용 |
| `/usage` | 요금제 한도, 남은 양, 속도 제한 상태 |
| `/stats` | 일별 토큰 사용량, 세션 수, 사용 패턴 히스토리 |

**VS Code**: 하단 상태바에 실시간 컨텍스트 사용률 항상 표시

### Claude Desktop

- 대화 입력창 근처에 **토큰 사용량 표시**
- 컨텍스트 한계에 가까워지면 **경고 표시**
- Settings → Usage에서 **상세 사용 기록** 확인

## 핵심 관리 명령어

### `/compact` — 대화 압축

```
> /compact                          # 자동 요약
> /compact API 변경에 집중해줘       # 특정 주제에 집중하여 압축
```

대화 내용만 요약 → 공간 확보. CLAUDE.md와 Auto Memory는 디스크에서 다시 로드 (사라지지 않음).

### `/clear` — 전체 초기화

```
> /clear
```

모든 대화 기록을 삭제합니다. 작업 전환 시 사용하세요.

### `/btw` — 컨텍스트에 남지 않는 사이드 질문

```
> /btw 이 함수 뭐하는 거야?
```

메인 대화에 영향 없이 빠른 질문만 던집니다.

---

## 자동 압축 (Auto-Compaction)

컨텍스트가 한계에 근접하면 Claude가 **자동으로 대화를 요약**합니다. 사용자 개입 불필요. `/compact`로 미리 압축하면 더 정확하게 관리할 수 있습니다.

---

## Auto Memory (자동 메모리)

Claude가 세션 간에 **배운 내용을 자동으로 저장**합니다.

- **저장 위치**: `~/.claude/projects/<project-name>/memory/`
- **매 세션 시작 시**: MEMORY.md 첫 200줄 자동 로드
- **확인/편집**: `/memory` 명령어

## 컨텍스트 절약 실전 팁

### 기본 습관

| 상황 | 명령어 | 이유 |
|------|--------|------|
| 작업 전환 | `/clear` | 이전 맥락이 새 작업 방해 방지 |
| 긴 작업 중간 (2시간+) | `/compact` | 공간 확보 |
| 빠른 질문 | `/btw` | 메인 흐름 보호 |
| 느려진 느낌 | `/context` | 사용량 진단 |

### 큰 파일 주의

큰 PDF, CSV, 로그 파일 등을 통째로 읽히면 컨텍스트가 한 번에 소진됩니다. **꼭 필요하지 않다면 지양하세요.**

| 상황 | 권장 방법 |
|------|-----------|
| 100페이지 PDF 분석 | 필요한 페이지만 지정: "3~5페이지만 읽어줘" |
| 대용량 CSV 분석 | 처음 100행만 먼저 확인 → 필요한 컬럼/조건 지정 |
| 긴 로그 파일 | "에러가 포함된 줄만 찾아줘" 등 조건 지정 |
| 큰 코드 파일 | 전체가 아니라 특정 함수/섹션 지정 |

**원칙**: "전체를 읽어줘"보다 "이 부분만 읽어줘"가 항상 낫습니다.

### 기타 팁

- **정기적으로 `/context` 확인**: 사용량 모니터링
- **작업 단위로 세션 분리**: 큰 작업은 여러 세션으로 나누기

## 상황별 체크리스트

### 새 프로젝트 시작
```
claude → /init → /context → 작업 시작
```

### 긴 세션 (2시간 이상)
```
/context → /compact (필요시) → 계속 작업
```

### 다른 프로젝트로 전환
```
/clear → cd 다른 프로젝트 → claude → 새 작업 시작
```

### 느려졌을 때
```
/context (진단) → /compact (압축) → 여전히 느리면 /clear (초기화)
```

## 자주 묻는 질문

**Q: 컨텍스트가 꽉 찼다는 경고가 나왔어요.**
A: `/context`로 현황 확인 → `/compact`로 압축. 여전히 꽉 차면 `/clear`로 초기화.

**Q: `/compact`와 `/clear`의 차이는?**
A: `/compact`는 대화를 요약해서 공간 확보 (작업 계속). `/clear`는 모든 대화 삭제 (새로 시작).

**Q: 세션을 닫았다가 다시 열면 이전 내용이 사라지나요?**
A: `claude -c`로 시작하면 직전 대화를 이어갈 수 있습니다. 그냥 `claude`로 시작하면 새 세션입니다.

**Q: Auto Memory에 뭐가 저장돼요?**
A: 프로젝트별로 배운 정보 (구조, 규칙, 자주 쓰는 패턴). `/memory`로 확인 가능.

**Q: 사용량 한도에 도달하면?**
A: 속도가 느려지거나 일시적으로 사용이 제한됩니다. `/usage`로 남은 한도를 확인하세요. Teams/Enterprise 계정은 관리자에게 문의.
